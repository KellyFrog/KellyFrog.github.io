NOI Linux 使用指南

本文将介绍 Linux(NOI Linux) 终端的一些基本语法。

  1. cd

cd 命令是在终端下进入某个文件夹，用法是 cd 路径，比如：

  1. ls / la

ls / la 命令是将当前文件夹内的所有文件列出来，其中 la 会显示隐藏的文件（不过 OI 上并没有什么特别的作用）

  1. mkdir

mkdir 命令是在终端下在当前文件夹下创建子文件夹，用法是 mkdir 文件夹名。

  1. cmp

cmp 命令是全文比较两个文件的，如果完全一致则不输出，否则输出第一个失配的字节。

值得一提的是，如果你从 c++ 中的 system() 函数执行此命令，你会得到一个返回值，如果匹配则为 0，否则不为 0。

  1. diff

用法和 cmp 类似，只不过是全文比较，会输出所有失配的地方，一般 OI 中有用的参数只有 -a、-i。

事实上，同 cmp，用 system() 调用一样会有返回值，应该和 cmp 一致。

  1. clear

清屏。

  1. g++ 编译

基本用法：g++ -o 执行文件名 源文件名 参数，意将 源文件名（必须是.cpp）通过附加 参数 的选项，编译为 执行文件名。

或者直接 g++ 源文件名 参数，将 源文件名 编译为 a.out 的可执行文件。

Linux 下的可执行文件的扩展名不一定是 .exe

一般有以下有用的参数。

参数 内容
-std=c++11/14/17 用 c++11/14/17 标准编译（NOI Linux 上的 g++ 4.8.4 只支持 c++11）
-O2 开启 O2 优化
-Wall 开启全部警告（如未使用变量、未返回的函数等）
-lm 未知

需要注意的是，测评时如果没有特殊说明，是没有任何优化（参数）的。

  1. 在终端下运行可执行文件

基本用法：./文件名 (< input) (> output)，其中 文件名 是可执行文件名，后面如果有 < input 就是将 input 文件作为 stdin 读入，否则从终端读入 stdin；同理，如果有 > output 则是将 stdout 输出到 output，否则直接输出到终端。

  1. Tab 键的自动补全

在输入命令的时候，Tab 键会帮助我们自动联想，如果只有一个合法选项，则会直接补全，建议多尝试尝试。

附：c++ 中 system() 的用法和对拍器。

system() 定义为 int system(const char *)，中间的是一个字符串，表示需要执行的命令，语法同上面的终端语法，之后会有一个范围值，为命令执行的返回值。

题解 P2605 【[ZJOI2010]基站选址】

感觉没有黑题的难度吧…….

我们首先考虑一个朴素的 DP，一个很显然的想法就是让 $dp_{i,j}$ 表示前 $i$ 个位置，放了 $j$ 个基站，补偿仅考虑下标 $[1,i]$ 中的点的最小花费。

这样我们再从 $i’$ 转移到 $i$ 的时候，再统计下标在 $[i’+1,i-1]$ 中的点的补偿就可以了，不难列出转移方程：

$$
dp_{i,j}=\min_{1\le i’<i}{dp_{i’,j-1}+\operatorname{cost}(i’+1)}+c_i
$$

其中 $\operatorname{cost}(i’+1)$ 表示在下标在 $[i’+1,i-1]$ 中的点需要支付的补偿和。

如果我们直接在转移的时候暴力 $O(n)$ 求解 $\operatorname{cost}$ 的话，总的复杂度是 $O(n^2k)$ 的，无法通过。

我们不难发现转移的瓶颈是在求解 $\operatorname{cost}$ 上的，我们不妨将其和 $dp_{i’,j-1}$ 一起看待，再采用某种方法的在转移的过程中动态维护最小值。

我们不妨令最左边（$dis$ 值最小的）能够覆盖到 $i$ 点的基站位置为 $L_i$，同理最右边（$dis$ 值最大的）能够覆盖到点 $i$ 的基站位置为 $R_i$。那么如果我们在下标在 $[L_i,R_i]$ 中的任意一个点放置了基站，点 $i$ 将被覆盖，不会产生补偿。

我们考虑一个 $L_i,R_i$，在什么时候他会对 $dp_{i’,j-1}+\operatorname{cost}(i’+1)$ 的值产生贡献呢？一定是当 $[L_i,R_i]$ 这个区间被 $[i’+1,i-1]$ 这个区间包含的时候会产生贡献。再进一步，当对 $[R_i+1,n]$ 中的 $i$ 进行转移时，会对 $[1,L_i-1]$ 中的 $i’$ 产生影响。

由于我们是顺序枚举 $i$，所以我们只需要在 $i=R_i$ 完成转移以后，将 $[1,L_i-1]$ 中所有 $i’$ 的 $\operatorname{cost}$ 加上 $W_i$ 即可，相当于我们动态维护了 $\operatorname{cost}$ 的值。

我们需要区间加、区间最小值，线段树即可。

需要注意的是最后答案并不是 $dp_{n,k}$，而应该是 $\min_{1\le i\le n}dp_{i,k}+\operatorname{cost}(i)$。

总的复杂度 $O(nk\log n)$。
