题解 P2605 【[ZJOI2010]基站选址】

感觉没有黑题的难度吧…….

我们首先考虑一个朴素的 DP，一个很显然的想法就是让 $dp_{i,j}$ 表示前 $i$ 个位置，放了 $j$ 个基站，补偿仅考虑下标 $[1,i]$ 中的点的最小花费。

这样我们再从 $i’$ 转移到 $i$ 的时候，再统计下标在 $[i’+1,i-1]$ 中的点的补偿就可以了，不难列出转移方程：

其中 $\operatorname{cost}(i’+1)$ 表示在下标在 $[i’+1,i-1]$ 中的点需要支付的补偿和。

如果我们直接在转移的时候暴力 $O(n)$ 求解 $\operatorname{cost}$ 的话，总的复杂度是 $O(n^2k)$ 的，无法通过。

我们不难发现转移的瓶颈是在求解 $\operatorname{cost}$ 上的，我们不妨将其和 $dp_{i’,j-1}$ 一起看待，再采用某种方法的在转移的过程中动态维护最小值。

我们不妨令最左边（$dis$ 值最小的）能够覆盖到 $i$ 点的基站位置为 $L_i$，同理最右边（$dis$ 值最大的）能够覆盖到点 $i$ 的基站位置为 $R_i$。那么如果我们在下标在 $[L_i,R_i]$ 中的任意一个点放置了基站，点 $i$ 将被覆盖，不会产生补偿。

我们考虑一个 $L_i,R_i$，在什么时候他会对 $dp_{i’,j-1}+\operatorname{cost}(i’+1)$ 的值产生贡献呢？一定是当 $[L_i,R_i]$ 这个区间被 $[i’+1,i-1]$ 这个区间包含的时候会产生贡献。再进一步，当对 $[R_i+1,n]$ 中的 $i$ 进行转移时，会对 $[1,L_i-1]$ 中的 $i’$ 产生影响。

由于我们是顺序枚举 $i$，所以我们只需要在 $i=R_i$ 完成转移以后，将 $[1,L_i-1]$ 中所有 $i’$ 的 $\operatorname{cost}$ 加上 $W_i$ 即可，相当于我们动态维护了 $\operatorname{cost}$ 的值。

我们需要区间加、区间最小值，线段树即可。

需要注意的是最后答案并不是 $dp_{n,k}$，而应该是 $\min_{1\le i\le n}dp_{i,k}+\operatorname{cost}(i)$。

总的复杂度 $O(nk\log n)$。

题解 CF1416C 【XOR Inverse】

用一种类似 CDQ 的思路（比赛的时候看有人写了 CDQ）。

对于当前的一个数列，我们先讨论最高位，然后递归计算接下来的低位。

对于逆序对，有这么几种：

  • 当前位都是 0 或 都是 1 的逆序对 ——这个我们递归调用
  • 当前位有 0 有 1 的逆序对——显然我们只关心这一位的大小关系，因为当前位是 0 或 1 的内部逆序对会被上面一种情况讨论到

如果我们这一位 $\operatorname{xor}$ 了一个 1，那么原来有 0 有 1 的顺序对就会变成逆序对，所以我们分别做顺序对和逆序对存起来最后比一下就行了。

题解 CF1343E 【Weights Distributing】

第一场 CF 的题为啥我这题都不会做啊。

题意（这鬼翻译居然是我写的）：

给每条边安排一个边权，最小化 $a$ 到 $b$ 和 $b$ 到 $c$ 的最短路长度和。

我们将 $a\rightarrow b$ 拆分为 $a\rightarrow p\rightarrow b$，同理 $c\rightarrow b$ 拆为 $c\rightarrow p\rightarrow b$，意思是 $a,c$ 两点在 $p$ 汇合之后走到 $b$，之前都没有汇合。

那么此时的答案就是 $2\cdot dis(b\rightarrow p)+dis(a\rightarrow p)+dis(c\rightarrow p)$，于是我们先以边权为 1，跑 $a,b,c$ 分别为源点的最短路，之后枚举 $p$ 点就行了（把尽量小的权值安排给 $b\rightarrow p$）。

但这样你发现过不了第一个样例，因为会有 $dis(a\rightarrow p)+dis(b\rightarrow p)+dis(c\rightarrow p) > m$，边权安排不过来，根据抽屉原理，一定会有一条边被经过两次，和上面“没有汇合”就矛盾了，于是我们舍去这种情况。

题解 CF1163F 【Indecisive Taxi Fee】

为了方便，我们用 $u\rightarrow v$ 表示 $u$ 到 $v$ 的最短路，$dis(u,v)$ 表示 $u$ 到 $v$ 的最短路长度。

我们先建出“最短路树”$G’$，这棵树上包含了原图上 $1$ 到任意一个点的最短路上的边，如果有多条最短路，我们只记录一条。

怎么求出最短路树呢？我们再跑最短路的时候记录一个点 $u$ 是从那个点走过来的，然后在树上建这么一条边。

接下来我们分类讨论修改的操作：

  • 如果修改的边 $u-v $ 在 $1\rightarrow n$ 上，并且将这条边的权值变小了，答案显然就是 $dis(1,n)-$ 减小的边权。
  • 如果修改的边 $u-v $ 在 $1\rightarrow n$ 上，并且这条边的边权变大了，这种情况比较复杂，我们后面单独讨论。
  • 如果修改的边 $u-v $ 不在 $1\rightarrow n$ 上，并且这条边的边权变小了，答案显然是 $\min(dis(1,n),\min(dis(1,u)+dis(v,n),dis(1,v)+dis(u,n))+$ 这条边的权值 $)$。
  • 如果修改的边 $u-v $ 不在 $1\rightarrow n$ 上，并且这条边的边权变大了，显然不会对最短路产生任何影响，答案就是 $dis(1,n)$。

（对于上例，我们所说的 $1\rightarrow n$ 是在最短路树上的唯一那条路径）

那么我们考虑如何维护第二种情况。

如果我们还按照原来 $1\rightarrow n$ 走，最短路长度就是 $dis(1,n)+$ 增加的权值，那么对于剩下的情况，考虑走 $1\rightarrow n$ 外的一条边权为 $w’$ 的边 $u’-v’$，其最短路长度为 $dis(1,u’)+w’+dis(v’,n)$。

我们令 $dis’(u)$，表示断掉最短路上 $u$ 和其父亲的边后能得到的最短路。

我们考虑 $1\rightarrow n$ 外的边权为 $w’$ 的边 $u’-v’$ 会对 $G’$ 上 $\operatorname{lca}(u’,n)\rightarrow \operatorname{lca}(v’,n)$ 上的 $dis’$ 有影响（我们假设 $\operatorname{lca}(u’,n)$ 的深度比 $\operatorname{lca}(v’,n)$ 更小）为什么是这样呢？

首先，是 $1\rightarrow \operatorname{lca}(u’,n)$，这么走一定是最短的，如果有其他 $1\rightarrow u’$ 的路径也是最短路，一定会有不在树上的边，我们会通过这些非树边更新 $1\rightarrow \operatorname{lca}(u’,n)$，同样，我们会在 $\operatorname{lca}(u’,n)$ 之后走和树上 $1\rightarrow n$ 不一样的一条路，最早的和 $1\rightarrow n$ 的汇合点会是 $\operatorname{lca}(v’,n)$，同样地，如果不汇合的话，一定会走别的边，这样 $\operatorname{lca}(v’,n)\rightarrow n$ 就会被别的边更新。

换句话说，我们对于每条边，只更新一定会被更新的边，如果在两个 $\operatorname{lca}$ 之后没有走树边反而更短，更短路径上的边将会继续更新树上的信息，从而保证正确性。

更新是取 $\min$，于是我们用线段树维护。总的复杂度 $O(n\log n+m\log n+q\log n)$。

细节较少，想明白了都是板子，但 20min AC 的就很离谱，我调了一个下午…

NOI Linux 使用指南

本文将介绍 Linux(NOI Linux) 终端的一些基本语法。

  1. cd

cd 命令是在终端下进入某个文件夹，用法是 cd 路径，比如：

  1. ls / la

ls / la 命令是将当前文件夹内的所有文件列出来，其中 la 会显示隐藏的文件（不过 OI 上并没有什么特别的作用）

  1. mkdir

mkdir 命令是在终端下在当前文件夹下创建子文件夹，用法是 mkdir 文件夹名。

  1. cmp

cmp 命令是全文比较两个文件的，如果完全一致则不输出，否则输出第一个失配的字节。

值得一提的是，如果你从 c++ 中的 system() 函数执行此命令，你会得到一个返回值，如果匹配则为 0，否则不为 0。

  1. diff

用法和 cmp 类似，只不过是全文比较，会输出所有失配的地方，一般 OI 中有用的参数只有 -a、-i。

事实上，同 cmp，用 system() 调用一样会有返回值，应该和 cmp 一致。

  1. clear

清屏。

  1. g++ 编译

基本用法：g++ -o 执行文件名 源文件名 参数，意将 源文件名（必须是.cpp）通过附加 参数 的选项，编译为 执行文件名。

或者直接 g++ 源文件名 参数，将 源文件名 编译为 a.out 的可执行文件。

Linux 下的可执行文件的扩展名不一定是 .exe

一般有以下有用的参数。

参数 内容
-std=c++11/14/17 用 c++11/14/17 标准编译（NOI Linux 上的 g++ 4.8.4 只支持 c++11）
-O2 开启 O2 优化
-Wall 开启全部警告（如未使用变量、未返回的函数等）
-lm 未知

需要注意的是，测评时如果没有特殊说明，是没有任何优化（参数）的。

  1. 在终端下运行可执行文件

基本用法：./文件名 (< input) (> output)，其中 文件名 是可执行文件名，后面如果有 < input 就是将 input 文件作为 stdin 读入，否则从终端读入 stdin；同理，如果有 > output 则是将 stdout 输出到 output，否则直接输出到终端。

  1. Tab 键的自动补全

在输入命令的时候，Tab 键会帮助我们自动联想，如果只有一个合法选项，则会直接补全，建议多尝试尝试。

附：c++ 中 system() 的用法和对拍器。

system() 定义为 int system(const char *)，中间的是一个字符串，表示需要执行的命令，语法同上面的终端语法，之后会有一个范围值，为命令执行的返回值。
